# TrabalhoIndividualTeoriaPratica

1. Defina POO:
	Programação Orientada a Objeto constitui na divisão e organização de tarefas utilizando objetos para conectar as informações entre classes e métodos.

2. Defina Classe:
	Classes possuem atributos e métodos próprios que a definem, podendo se relacionar ou não com outras classes a partir de sua tipagem, sendo extremamente semelhante a uma estrutura.

3. Defina método:
	Métodos são as operações que ocorrem dentro de uma classe ou estrutura.

4. Defina Herança e dê exemplos:
	Heranças são classes que possuem um sistema de recebimento de atributos e métodos da classe originária, podendo ser chamada de classe pai ou mãe e classe filha. Por exemplo no cadastro de Pessoa Física e Jurídica, possuindo características comuns entre elas como nome e data de nascimento, com isso pode-se ser criado uma classe Pessoa possuindo esses dados comuns sendo ela a classe "mãe".

5. Defina Composição e dê exemplos:
	Enquanto a Herança utiliza um sistema hierárquico, usando a classe Base inteiramente e sendo sensível nas modificações da mesma, na composição utiliza-se um sistema de "ajuda", como por exemplo um trabalho cecíproco, a partir de requerimentos pré impostos a classe pode ser executada, podendo possuir procedimentos e funções na mesma base sem afetar sua funcionalidade.

6. O que é um parâmetro e qual a diferença para um argumento?
	Um parâmetro são pré requisitos para a utilização de um método, sendo diferente de um argumento, que é o dado a ser passado para o método.

7. O que é sobrecarga de métodos?
	São métodos com parâmetros pré estabelecidos, que mesmo se dois métodos possuirem o mesmo nome com parâmetros diferentes os argumentos definirão o destinatário.

8. O que são parâmetros opcionais?
	São parâmetros com um dado default, para caso o campo não receba os dados do usuário o programa execute e funcione.

9. O que são parâmetros nomeados?
	São identificadores de destinatário para os dados, por exemplo um método possui os parâmetros Id, Nome e Idade, e para levar os dados a preencher ester campos vindos de fora da classe pode-ser identificar o campo e o dado a preencher ele ao lado.
